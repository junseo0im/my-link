# [PRD] 링크트리 클론 서비스 "마이링크 (MyLink)" 기능정의서

본 문서는 개인의 다중 링크를 하나의 아름다운 랜딩 페이지로 연결하는 '원링크(One-Link)' 플랫폼 **"마이링크 (MyLink)"**의 제품 기능 정의서(PRD)입니다. 

---

## 1. 프로젝트 개요

| 항목 | 상세 내용 |
| :--- | :--- |
| **프로젝트명** | 마이링크 (MyLink) |
| **서비스 목적** | 인스타그램, 유튜브, 블로그, 포트폴리오 등 여러 플랫폼에 분산된 개인의 활동 링크를 단 하나의 모바일 친화적인 랜딩 페이지(`mylink.com/username`)로 결합합니다. 사용자가 자신을 효과적으로 셀프 브랜딩하고, 채널 트래픽을 극대화할 수 있도록 돕습니다. |
| **대상 사용자** | - **크리에이터/인플루언서**: 소셜 미디어 프로필 링크에 자신들의 모든 활동 채널을 모아놓고자 하는 사용자<br>- **프리랜서/개발자/디자이너**: 자신의 포트폴리오, 이력서, 연락처 등을 한눈에 보여주고자 하는 전문가<br>- **1인 기업가/소상공인**: 스마트스토어, 공식 홈페이지, 문의 창구 등을 깔끔하게 통합하여 비즈니스 효율을 높이려는 사용자 |

### 1.2 기술 스택 및 디자인 시스템

| 구분 | 도입 기술 | 비고 |
| :--- | :--- | :--- |
| **Front-End** | React / Next.js | 빠른 렌더링 및 SEO 대응을 위해 Next.js 권장 |
| **Back-End** | Firebase (Firestore, Auth) | 나중 이식 예정 (Phase 1 이후 연동) |

| **디자인 시스템** | **shadcn/ui** (Radix UI + TailwindCSS) | 접근성과 일관된 UI 구현을 위한 컴포넌트 라이브러리 |
| **스타일링** | Vanilla CSS / TailwindCSS | shadcn/ui 구동 및 신속한 스타일 커스텀 |

## 2. 핵심 기능 목록 (필수 vs 선택 vs 향후 구현)

마이링크 서비스는 점진적 개발 모델을 따르며, 초기 구축 단계(Phase 1)부터 추후 확장 단계(Phase 2)까지 유기적으로 확장될 수 있도록 설계합니다.

### 2.1 필수 기능 (Phase 1)
1. **구글 소셜 로그인 및 회원가입** (Firebase Authentication 연동 예정)
2. **개인화 동적 URL 라우팅** (`mylink.com/username` 형식으로 본인 랜딩 페이지 표출)
3. **프로필 기본 관리** (프로필 사진 업로드, 타이틀 설정, 한 줄 설명 작성)
4. **링크 목록 관리 (CRUD)** (링크 제목, URL 추가/수정/삭제 및 토글로 활성화/비활성화)
5. **드래그 앤 드롭 링크 순서 변경** (관리자 화면에서 직관적인 링크 순서 배치)
6. **기본 디자인 테마 적용** (사전 정의된 8종의 컬러 테마 및 배경 그라디언트 제공)
7. **기본 SEO 최적화 설정** (개인 랜딩 페이지별 메타 타이틀, 메타 설명 설정)

### 2.2 선택 및 고도화 기능 (Phase 1.5)
1. **커스텀 디자인 세부 조정** (버튼의 라운드 처리, 테두리 스타일, 그림자 효과, 웹 폰트 선택, 마우스 호버 애니메이션 효과 등)
2. **미디어 및 소셜 위젯 임베드** (단순 텍스트 링크를 넘어 유튜브 비디오, 사운드클라우드 플레이어 등을 페이지 내부에서 직접 감상할 수 있는 컴포넌트)

### 2.3 향후 구현 기능 (Phase 2)
1. **방문자 통계 및 대시보드** (방문자 수(PV/UV), 링크별 클릭률(CTR), 유입 기기(모바일/데스크톱), 유입 소스 분석)

## 3. 기능 상세 명세 (Functional Specifications)

### 3.1 회원가입 및 구글 소셜 로그인
- **기능 목적**: 사용자 고유 식별 및 본인 소유의 마이링크 관리 대시보드 권한 부여.
- **상세 스펙**:
  - Firebase Authentication의 Google OAuth 2.0 제공업체를 연동하여 원클릭 로그인을 지원합니다.
  - 로그인 성공 시, 사용자 정보가 최초 로그인인 경우 사용자 DB에 자동 등록됩니다.
  - 회원가입 프로세스 중 **고유 사용자명(Username)** 설정을 요구하여, 중복 검사 후 개인 링크 주소(`mylink.com/username`)로 매핑합니다.
  - 비로그인 사용자는 다른 사람의 마이링크 프로필 페이지만 조회할 수 있습니다.

### 3.2 개인화 동적 URL 라우팅 및 프로필
- **기능 목적**: 사용자의 닉네임을 기반으로 하는 편리한 주소를 제공하며, 해당 주소 접속 시 모바일 반응형 프로필 카드를 렌더링합니다.
- **상세 스펙**:
  - **닉네임 기반 라우팅 (핸들 배제)**: 별도의 고유 핸들(아이디) 등록 단계 없이, 사용자가 설정한 **닉네임(displayName)**을 주소 경로로 바로 사용합니다. (`mylink.com/[displayName]` 혹은 `mylink.com/profile/[displayName]`)
    - *예*: 닉네임이 `홍길동`인 경우 `mylink.com/홍길동`으로 본인의 프로필 카드가 렌더링됩니다.
  - **닉네임 고유성(Unique) 검증**:
    - URL 충돌을 방지하기 위해 `displayName`은 시스템 내에서 **단 하나만 존재(Unique)**해야 합니다.
    - 최초 가입 및 닉네임 수정 시 DB 내 중복 여부를 실시간으로 조회하여 중복된 닉네임은 설정을 제한합니다.
  - **닉네임 유효성 검사 (URL-Safe 규격)**:
    - 주소 표시 한계와 깨짐을 방지하기 위해 닉네임에는 **영문 대소문자, 숫자, 한글(자모음 완성형), 하이픈(`-`), 언더바(`_`)**만 허용합니다.
    - 공백(띄어쓰기) 및 특수문자(`?`, `/`, `#`, `&` 등 URL 예약어)는 입력이 제한됩니다. (정규식: `^[a-zA-Z0-9ㄱ-ㅎㅏ-ㅣ가-힣-_]+$`)
  - **프로필 및 닉네임 수정 및 릴리즈 규칙**:
    - 사용자는 대시보드에서 **닉네임(displayName)**과 **소개글(bio)**을 실시간 수정 가능합니다.
    - 닉네임 수정 시 기존 URL은 즉시 해제(Release)되어 타인이 바로 사용할 수 있도록 릴리즈되며, 새로운 URL 경로가 바인딩됩니다.
    - *UX 안내*: 닉네임 수정 클릭 시 "닉네임을 변경하면 기존 링크 주소(`mylink.com/기존닉네임`)는 즉시 연결이 끊어지며 복구할 수 없습니다."라는 경고 확인 팝업을 노출하여 실수를 방지합니다.
  - **이미지 업로드 배제 및 프로필 이미지 처리**: 
    - 서버 부하와 복잡한 스토리지 설정을 줄이기 위해 **파일 업로드 기능은 배제**합니다.
    - 대신, **구글 소셜 로그인 시 받아오는 프로필 이미지 URL**(`photoURL`)을 기본 연동하여 띄웁니다.
    - 구글 프로필 이미지가 없거나 미연동 시, 닉네임의 첫 글자(초성/이니셜)를 활용한 텍스트 기반 아바타(shadcn/ui Avatar Fallback 사용)로 표시합니다.
  - **프로필 카드 구성**:
    - 중앙 정렬 레이아웃 (모바일 세로형 레이아웃 기본 채택)
    - **우측 상단 공유 위젯**: 원클릭으로 본인의 마이링크 URL을 복사할 수 있는 '링크 복사 아이콘' 및 모바일 OS 공유 시트를 띄우는 '공유하기' 플로팅 버튼 배치
    - 원형 프로필 이미지 (구글 연동 이미지 또는 이니셜 아바타)
    - **표시 이름 (displayName)**: 프로필 메인 타이틀로 노출될 실명 혹은 닉네임 (최대 30자)
    - **소개글 (bio)**: 프로필 소개글 (최대 80자)
    - 하단에 등록된 활성 링크 버튼 목록 나열

### 3.3 링크 목록 관리 및 순서 변경 (CRUD)
- **기능 목적**: 사용자가 대시보드에서 자신의 개인 랜딩 페이지에 노출할 링크들을 자유롭게 설정하고 관리합니다.
- **상세 스펙**:
  - **인라인 편집 (Inline Editing)**:
    - 링크를 수정하기 위해 별도의 수정 모달이나 페이지로 이동할 필요 없이, 대시보드 화면상에서 링크의 **제목**과 **URL** 텍스트 영역을 클릭하면 즉시 입력 필드(Input)로 전환되어 현장에서 바로 수정할 수 있는 **인라인 편집**을 적용합니다.
    - 입력 필드 포커스가 해제(blur)되거나 엔터(Enter) 키 입력 시 실시간으로 DB에 저장됩니다.
  - **파비콘 자동 로딩 (Google Favicon API)**:
    - 사용자가 직접 링크 썸네일 이미지를 업로드하는 대신, 입력된 URL 도메인 주소로부터 파비콘을 자동으로 파싱하여 보여줍니다.
    - 이때, **Google Favicon API**를 활용하여 매우 빠르고 가벼운 속도로 파비콘 이미지를 받아옵니다.
    - *API 엔드포인트 예시*: `https://www.google.com/s2/favicons?domain=[domain_name]&sz=64`
  - **링크 데이터 모델**:
    ```typescript
    interface LinkItem {
      id: string;
      title: string;       // 사용자 노출용 링크 제목 (예: "내 포트폴리오 보러가기")
      url: string;         // 실제 이동할 대상 URL (http:// 또는 https:// 필수 검증)
      isActive: boolean;   // 랜딩 페이지 노출 여부 토글
      order: number;       // 링크 정렬 순서
      faviconUrl?: string; // Google Favicon API로 생성된 파비콘 URL
    }
    ```
  - **드래그 앤 드롭 정렬**: 관리자 화면에서 `react-beautiful-dnd` 등의 라이브러리를 활용해 터치/드래그 조작으로 링크들의 `order` 값을 실시간 변경하여 화면에 즉시 반영되도록 구현합니다.
  - **유효성 검사**: URL 입력 시 올바른 도메인 형식인지 정규식 검사를 실시합니다.

### 3.4 디자인 커스터마이징 및 디자인 시스템 (shadcn/ui)
- **기능 목적**: 사용자의 취향과 퍼스널 브랜드 이미지에 맞는 랜딩 페이지 디자인 환경을 제공하며, 개발 생산성과 일관된 사용자 경험(UX)을 위해 UI 컴포넌트 표준을 구축합니다.
- **디자인 시스템 적용 사양 (shadcn/ui & Radix UI)**:
  - **기반 기술**: Headless UI 라이브러리인 Radix UI와 TailwindCSS의 결합 체계인 `shadcn/ui`를 도입합니다.
  - **테마 바인딩**: shadcn/ui의 CSS 변수 구조(`--background`, `--primary`, `--radius` 등)를 활용하여 사용자가 테마를 변경할 때 실시간으로 이 변수값을 동적 매핑하여 커스텀 스타일이 반영되도록 설계합니다.
- **상세 스펙**:
  - **기본 테마 (Phase 1)**: 단색(클래식 화이트, 미드나잇 블랙), 그라디언트 테마(선셋 퍼플, 오션 블루, 에메랄드 민트 등)를 포함한 프리셋 테마 8종을 제공하여 원클릭으로 페이지 전체 스타일을 적용합니다.
  - **커스텀 디자인 세부 조정 (Phase 1.5)**:
    - **버튼 스타일**: 테두리 없음(Flat), 둥근 모서리(Rounded), 완전 둥글게(Pill), 테두리만 있는 스타일(Outline), 그림자 효과(Soft Shadow). 
      - *구현 스펙*: shadcn/ui `Button` 컴포넌트의 `variant` 속성(default, outline, ghost) 및 TailwindCSS의 보더 반경(`rounded-md`, `rounded-full` 등) 조정을 연계합니다.
    - **글꼴**: 기본 시스템 폰트 외에 Google Fonts 연동 (Inter, Roboto, Outfit, Noto Sans KR 등 5종 선택 가능).
    - **애니메이션**: 특정 중요한 링크에 주의를 끌 수 있도록 마우스 호버 또는 무한 반복 애니메이션 효과(바운스 Bounce, 흔들림 Wiggle, 깜빡임 Flash 등)를 링크 아이템 개별로 적용할 수 있습니다. 
      - *구현 스펙*: TailwindCSS `animate` 클래스 확장 또는 `framer-motion` 패키지를 통해 shadcn 컴포넌트에 결합합니다.

### 3.5 미디어 및 소셜 위젯 임베드 (Phase 1.5)
- **기능 목적**: 외부 플랫폼의 콘텐츠를 이동 없이 마이링크 페이지 내부에서 직접 재생하게 하여 이탈률을 방지합니다.
- **상세 스펙**:
  - **유튜브 위젯**: 유튜브 비디오 URL을 입력하면, 단순 링크 버튼 대신 반응형 `iframe` 플레이어를 임베드하여 동영상을 바로 재생합니다.
  - **사운드클라우드/스포티파이 위젯**: 음원 플레이어 임베드를 지원하여 배경음악이나 팟캐스트를 바로 청취하게 합니다.
  - **인스타/X(트위터) 연동**: 프로필 바로 밑에 텍스트 아이콘 형태의 소셜 링크 모음(인스타그램, 유튜브, 깃허브, 페이스북, 이메일 등)을 일렬로 배치하는 위젯을 제공합니다.

### 3.6 개별 프로필 SEO 설정 (Phase 1)
- **기능 목적**: 사용자가 포털이나 소셜 네트워크(카카오톡, 슬랙 등)에 본인의 마이링크 주소를 공유했을 때 오픈그래프(OG) 이미지와 문구를 커스터마이징하여 전문적인 브랜딩을 돕습니다.
- **상세 스펙**:
  - **대시보드 내 SEO 탭 제공**:
    - **SEO Title**: 브라우저 탭과 검색 결과에 노출될 타이틀 (예: "개발자 홍길동 | 마이링크")
    - **SEO Description**: 검색 로봇 및 공유 링크 미리보기에 사용될 요약 글
    - **OG Image**: 카카오톡이나 페이스북 등에 공유 시 노출될 대표 썸네일 이미지 등록 (등록하지 않을 경우 기본 마이링크 로고 이미지 노출)

### 3.7 방문자 통계 분석 (Phase 2 - 향후 구현)
- **기능 목적**: 어떤 경로로 방문자가 유입되었고, 어떤 링크가 가장 반응이 좋았는지 정량적 데이터를 시각화하여 제공합니다.
- **상세 스펙**:
  - **수집 이벤트**:
    - `page_view`: 사용자 프로필 페이지 방문 시 기록 (중복 유입 방지를 위해 세션별 분석 적용)
    - `link_click`: 특정 링크 버튼을 클릭하여 외부 사이트로 연결되는 순간 이벤트 트래킹 (어떤 ID의 링크를 클릭했는지 기록)
  - **대시보드 통계 뷰**:
    - 금일/최근 7일/30일 기준 총 방문자 수(PV/UV) 그래프
    - 전체 링크 대비 각 링크의 클릭 횟수 및 클릭률(CTR) 순위 테이블
    - 기기 비율(Mobile vs Desktop) 및 주요 리퍼러(Instagram, YouTube, Direct 등) 비율 차트

### 3.8 데이터베이스 구조 및 데이터 모델링 (Firestore)
- **기능 목적**: 사용자 및 링크 데이터를 안정적으로 보관 및 조회하며, 대용량 트래픽 및 보안에 효율적인 구조로 설계합니다.
- **데이터베이스 모델 핵심 방향**:
  - **NoSQL Firestore**의 문서-컬렉션 구조를 적극적으로 활용합니다.
  - 각 사용자의 링크 목록은 루트 수준의 독립된 컬렉션이 아닌, 해당 사용자 문서 하위의 **서브컬렉션(Subcollection)**으로 설계하여 데이터의 소유 관계와 보안 규칙(Security Rules) 적용을 명확히 합니다.
- **Firestore 스키마 명세**:

#### 1) 루트 컬렉션: `users`
사용자의 가입 정보 및 프로필 설정을 저장하는 기본 문서군입니다.

- **경로**: `/users/{userId}`
- **필드 정의**:
  | 필드명 | 데이터 타입 | 설명 | 예시 |
  | :--- | :--- | :--- | :--- |
  | `uid` | string | Firebase Auth 고유 사용자 식별자 | `"u1W8x9J4..."` |
  | `displayName` | string | 사용자 표시 이름 (URL 경로 매핑 및 랜딩 페이지 최상단 표출) | `"홍길동"` |
  | `bio` | string | 한 줄 프로필 소개글 | `"Next.js와 테크를 좋아하는 프론트엔드 개발자입니다."` |
  | `googleProfileImageUrl` | string | 구글 소셜 로그인 시 연동되어 제공되는 프로필 이미지 주소 (업로드 배제) | `"https://lh3.googleusercontent.com/..."` |
  | `themeId` | string | 선택한 프리셋 디자인 테마 고유 번호/ID | `"theme-dark-01"` |
  | `seoSettings` | map | 오픈그래프 및 SEO 최적화 메타 설정 맵 | `{ title: "...", description: "..." }` |
  | `createdAt` | timestamp | 계정 생성 일시 | `2026-05-26 17:18:20` |

#### 2) 서브컬렉션: `links`
각 사용자가 등록한 링크 아이템들이 모여 있는 사용자의 하위 컬렉션입니다.

- **경로**: `/users/{userId}/links/{linkId}`
- **필드 정의**:
  | 필드명 | 데이터 타입 | 설명 | 예시 |
  | :--- | :--- | :--- | :--- |
  | `id` | string | 링크 문서 고유 식별 ID | `"link-abc-123"` |
  | `title` | string | 랜딩 페이지 버튼에 표시될 텍스트 (대시보드에서 인라인 편집 가능) | `"GitHub 바로가기"` |
  | `url` | string | 클릭 시 이동할 외부 URL 주소 (대시보드에서 인라인 편집 가능) | `"https://github.com/..."` |
  | `isActive` | boolean | 사용자 프로필에서의 링크 노출 여부 | `true` |
  | `order` | number | 링크 노출 정렬 순서 (오름차순 정렬) | `1` |
  | `faviconUrl` | string | Google Favicon API를 통해 도메인에서 자동 파싱된 파비콘 주소 | `"https://www.google.com/s2/favicons?domain=github.com&sz=64"` |
  | `animationType` | string | (선택) 강조 애니메이션 식별자 | `"bounce"` |
  | `updatedAt` | timestamp | 마지막 수정 시간 | `2026-05-26 17:18:20` |

### 3.9 데이터 보안 및 접근 권한 설계 (Firebase Security Rules)
- **기능 목적**: 악의적인 타인의 데이터 임의 위변조를 막고, 사용자 프라이버시를 안전하게 보호합니다.
- **상세 스펙**:
  - **인증 사용자 권한 제한**: 본인의 UID에 매핑된 데이터만 생성, 수정, 삭제(Write/Update/Delete)할 수 있도록 차단합니다.
  - **전체 조회 허용**: 외부 방문자 누구나 프로필 및 링크 데이터를 정상적으로 볼 수 있도록, 전체 문서 및 서브컬렉션에 대해 읽기(Read) 권한은 전체 공개로 지정합니다.
  - **Firestore 보안 규칙 가이드라인**:
    ```javascript
    rules_version = '2';
    service cloud.firestore {
      match /databases/{database}/documents {
        
        // 1) users 루트 컬렉션 권한 규칙
        match /users/{userId} {
          // 비로그인 사용자 포함 누구나 다른 사용자의 프로필 조회 가능
          allow read: if true;
          // 로그인한 당사자 본인만 프로필 등록 및 수정 가능
          allow write, update: if request.auth != null && request.auth.uid == userId;
          // 회원 탈퇴 등의 작업 시 본인만 문서 삭제 가능
          allow delete: if request.auth != null && request.auth.uid == userId;
        }

        // 2) users/{userId}/links 서브컬렉션 권한 규칙
        match /users/{userId}/links/{linkId} {
          // 누구나 사용자의 활성화된 링크 데이터 조회 가능
          allow read: if true;
          // 오직 링크의 실 소유주(본인)만 링크 CRUD(인라인 편집 포함) 가능
          allow write, update, delete: if request.auth != null && request.auth.uid == userId;
        }
      }
    }
    ```

## 4. 제안하는 추가 개선 아이디어 (마이링크 독창성 확보)

마이링크 서비스가 타사 서비스(링크트리 등)와 비교했을 때 확실한 강점을 가지기 위해 설계 단계에서 반영할 수 있는 추가 기능 제안입니다.

1. **링크 게시 예약 및 만료 설정 (선택 - Phase 1.5)**
   - **기능 설명**: 특정 이벤트, 공동 구매, 채용 공고 등 정해진 기간에만 링크를 띄워야 하는 사용자를 위해, 링크의 '자동 활성화 시작 일시' 및 '자동 비활성화 만료 일시'를 예약할 수 있는 기능입니다.
   - **가치**: 인플루언서들이 마케팅 캠페인을 야간이나 휴일에 수동으로 토글하지 않고 자동으로 관리할 수 있어 활용성이 뛰어납니다.
2. **민감한 콘텐츠 경고 기능 (필수/선택 - Phase 1)**
   - **기능 설명**: 성인용 콘텐츠, 의학적 조언, 혹은 폭력성이 있는 콘텐츠로 유도되는 링크의 경우, 클릭 시 "본 링크는 외부 페이지로 연결되며 민감한 내용을 포함할 수 있습니다. 계속하시겠습니까?"라는 팝업 경고를 띄우는 필터 시스템입니다.
   - **가치**: 구글 및 스토어 규정 준수 및 플랫폼의 보안/신뢰성을 위해 반드시 필요합니다.
3. **네이버페이 / 토스페이 송금 위젯 (선택 - Phase 2)**
   - **기능 설명**: 후원이나 1:1 서비스 결제를 유도하기 위해 토스 송금 링크나 네이버페이 QR 결제창으로 바로 넘어갈 수 있는 디자인 위젯을 프로필 하단에 배치합니다.
   - **가치**: 국내 사용자층(특히 프리랜서 및 크리에이터)의 실질적인 수익 창출 도구로 강력하게 작용할 수 있습니다.
