# [사용자 시나리오] 링크트리 클론 서비스 "마이링크 (MyLink)"

본 문서는 마이링크 서비스의 핵심 퍼소나인 **방문자(Visitor)**와 **소유자(Owner)**의 여정을 정의한 사용자 시나리오 문서입니다. 모든 시나리오 문장은 지정된 표준 문법 형태인 **"사용자는 [목적]하기 위해 [행동]한다"**에 맞추어 완벽하게 구조화되었습니다.

---

## 1. 방문자 (Visitor) 시나리오

방문자는 마이링크 소유자의 프로필 랜딩 페이지에 접속하여 정보를 소비하고 확산하는 주체입니다.

### 1.1 마이링크 프로필 카드 방문 및 조회
- 방문자는 **소유자의 퍼스널 브랜드 정보와 소셜 미디어 채널을 한눈에 파악하기 위해** 브라우저 주소창에 소유자의 닉네임이 반영된 주소(`mylink.com/닉네임`)를 입력하여 랜딩 페이지에 접속한다.
- 방문자는 **소유자가 작성한 공식적인 소개 글을 읽기 위해** 프로필 원형 이미지 하단에 정렬된 표시 이름(displayName)과 소개 문구(bio)를 차례대로 확인한다.

### 1.2 링크 클릭 및 외부 서비스 이동
- 방문자는 **소유자가 연결해 둔 포트폴리오나 소셜 플랫폼으로 안전하게 이동하기 위해** 개별 링크 버튼 좌측에 표시된 구글 API 파이프라인 연동 파비콘 아이콘을 확인하고 해당 버튼을 탭한다.
- 방문자는 **외부 사이트 이동 시 악성 페이지 유입이나 도메인 혼선을 예방하기 위해** 클릭 시 제공되는 대상 도메인 주소 정보를 확인하며 최종 브라우저 창으로 리다이렉트된다.

### 1.3 프로필 링크 복사 및 소셜 공유 (공유 위젯)
- 방문자는 **소유자의 마이링크 프로필 카드를 자신의 카카오톡 단체 채팅방이나 개인 소셜 미디어에 공유하기 위해** 프로필 우측 상단에 위치한 '공유하기' 플로팅 버튼을 클릭한다.
- 방문자는 **특정 커뮤니티나 텍스트 입력 영역에 수동으로 주소를 전달하기 위해** 공유 위젯 내 '링크 복사 아이콘'을 터치하여 클립보드에 해당 URL 주소를 저장한다.

### 1.4 존재하지 않는 닉네임 URL 유입 (404 예외 대응)
- 방문자는 **주소 입력 실수나 소유자의 닉네임 변경으로 깨진 주소에 도달했을 때 이탈하지 않고 정보를 얻기 위해** 404 커스텀 에러 화면("해당 마이링크를 찾을 수 없습니다")에 노출된 마이링크 서비스 소개를 읽는다.
- 방문자는 **자신만의 고유한 랜딩 페이지를 즉시 생성하기 위해** 404 오류 안내 페이지 하단에 배치된 "나만의 마이링크 만들기" 버튼을 눌러 서비스 가입 화면으로 즉시 전환한다.

### 1.5 민감한 콘텐츠 외부 링크 진입 및 경고
- 방문자는 **소유자가 성인 인증이나 주의 요함 태그를 걸어둔 외부 서비스에 접속하기 위해** 대상 링크 버튼을 우선 탭한다.
- 방문자는 **접속 전 유해성 또는 민감성 정보를 최종적으로 인지하고 승인하기 위해** 화면에 팝업되는 경고 메시지 창에서 "이해했으며 외부 사이트로 계속 이동합니다" 버튼을 누른다.

## 2. 소유자 (Owner) 시나리오

소유자는 자신만의 원링크 페이지를 구축하여 대시보드에서 프로필 및 링크를 관리하는 주체입니다.

### 2.1 최초 가입 및 고유 닉네임 설정
- 소유자는 **비밀번호 유출 걱정 없이 신속하고 안전하게 서비스를 시작하기 위해** 회원가입 화면에서 "구글 소셜 로그인" 버튼을 눌러 연동 인증을 마친다.
- 소유자는 **자신의 랜딩 페이지에 쓰일 고유의 URL 주소(`mylink.com/닉네임`)를 점유하기 위해** 가입 양식 창의 닉네임(displayName) 입력칸에 한글/영문 완성형 단어를 타이핑하고 실시간 중복 검사를 통과시킨다.

### 2.2 대시보드 진입 및 프로필 정보 확인
- 소유자는 **자신의 모바일 프로필에 보여질 최초 레이아웃 구성을 검토하기 위해** 가입 완료 후 자동으로 진입되는 관리 대시보드 메인 화면을 확인한다.
- 소유자는 **복잡한 파일 업로드 절차 없이 깔끔한 프로필 이미지를 유지하기 위해** 대시보드 내 원형 아바타 영역에 구글 소셜 계정의 이미지(`photoURL`)가 연동되었거나, 닉네임 첫 자 이니셜 아바타가 기본 활성화되었는지 확인한다.

### 2.3 링크 추가 (Create)
- 소유자는 **방문자들을 새로운 개인 블로그 글로 유도하기 위해** 대시보드의 "새 링크 추가" 영역에서 제목 입력 필드와 실제 도메인 URL 주소를 순차적으로 기입한다.
- 소유자는 **불완전하거나 형식이 깨진 잘못된 주소가 대중에게 공개되는 사고를 막기 위해** 도메인 양식 검증 규칙(`http://` 또는 `https://` 필수 체크)의 유효성 알림을 확인하고 최종 등록 처리를 마친다.

### 2.4 링크 내용 인라인 편집 (Update - Inline Editing)
- 소유자는 **기존에 등록된 링크의 맞춤법 오류나 세부 경로 변경을 복잡한 과정 없이 빠르게 교정하기 위해** 대시보드 목록에 나열된 대상 링크의 제목/URL 영역을 직접 더블클릭한다.
- 소유자는 **변경된 정보의 실시간 저장과 업데이트를 누락 없이 끝내기 위해** 텍스트 수정을 마친 후 입력 폼 바깥 영역을 마우스로 클릭(blur)하여 데이터베이스 자동 저장을 발동시킨다.

### 2.5 링크 순서 변경 (Update - Drag & Drop)
- 소유자는 **특정 시즌 이벤트나 가장 중요도가 높은 핵심 링크를 랜딩 페이지 최상단에 배치하기 위해** 대시보드에 구성된 개별 링크 카드 좌측의 드래그 핸들을 마우스로 쥔다.
- 소유자는 **방문자가 마이링크 진입 시 시각적 흐름에 맞춰 링크를 순차 탐색하도록 만들기 위해** 드래그 앤 드롭 조작을 통하여 직관적으로 정렬 순서(`order` 값)를 재조정한다.

### 2.6 링크 삭제 (Delete)
- 소유자는 **공동구매 기간이 지나 만료되었거나 더 이상 유효하지 않은 채널의 연동을 완전히 제거하기 위해** 해당 링크 카드의 우측 모퉁이에 위치한 "휴지통 아이콘(삭제)"을 터치한다.
- 소유자는 **중요한 링크 데이터가 실수로 즉시 지워져 복구하지 못하는 재앙을 막기 위해** 화면에 표시되는 "정말 이 링크를 삭제하시겠습니까?" 2차 확인 모달에서 최종 삭제 동의 버튼을 선택한다.

### 2.7 프로필/닉네임 수정 및 URL 릴리즈 처리
- 소유자는 **자신의 퍼스널 브랜딩 타이틀과 소개 요약 글을 변경하기 위해** 대시보드의 프로필 관리 설정 창 내에 있는 닉네임(displayName) 및 한 줄 소개(bio) 필드를 각각 갱신한다.
- 소유자는 **닉네임 변경 시 발생할 수 있는 기존 주소 단절에 대한 책임을 명확히 인지하기 위해** "변경 즉시 구 주소(`mylink.com/구닉네임`)는 소유권이 반환되어 누구나 등록할 수 있게 된다"는 팝업 경고 메시지를 상세히 검토한 뒤 변경을 최종 승인한다.
